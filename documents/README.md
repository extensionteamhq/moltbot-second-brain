# Documents Directory

This directory is the primary knowledge base for **Mateo Moore's Second Brain**. All `.md` files here (and in the sibling `journals/` and `briefs/` directories) are automatically served by the document browser at the Vercel-hosted app.

---

```

├── briefs/                    ← Daily morning briefs (auto-generated by Molly)
├── journals/                  ← Daily journal entries and session summaries
└── documents/
    ├── README.md              ← This file
    ├── dirt-roamers/          ← Land flipping business (dirtroamers.com)
    ├── anchor-staff/          ← Marketing & consulting agency (anchorandstaff.com)
    ├── partner-with-mateo/    ← Affiliate resources (partnerwithmateo.com)
    ├── rank-n-soar/           ← Rank & rent model (Payload CMS / Next.js sites)

```

## Folder Descriptions

Daily morning brief files generated by Molly. Filename format: `YYYY-MM-DD-brief.md`.
Required tag: `brief`

### `journals/` _(root level)_

Daily journal entries from Mateo's sessions with Molly. Includes both full journal entries (`YYYY-MM-DD-journal.md`) and daily summary notes (`YYYY-MM-DD.md`).
Required tag: `journal`

### `documents/dirt-roamers/`

All research, SOPs, guides, email sequences, and content for the **Dirt Roamers** land flipping business. Includes 30 numbered deep-research docs (01–29 + summary) and 30+ additional operational docs.
Required tag: `dirt-roamers`

### `documents/anchor-staff/`

SOPs, brand guides, media kit, intake questionnaires, nurture sequences, and AI tools digest for **Anchor & Staff** (full-service marketing/consulting agency with Mike Davis).
Required tag: `anchor-staff`

### `documents/partner-with-mateo/`

Affiliate resources, accountability guides, and materials for the **Partner With Mateo** community (SubTo / Gator / Top-Tier TC affiliate programs).
Required tag: `partner-with-mateo`

### `documents/rank-n-soar/`

Business plan (phases 0–8), financial models, expansion ideas, and SEO strategy for the **Rank-n-Soar** rank-and-rent website business.
Required tag: `rank-n-soar`

### `documents/mr-mateo-moore/`

Personal brand content — lead magnets, email campaigns, pricing guides, and marketing materials for **mrmateomoore.com**.
Required tag: `mr-mateo-moore`

### `documents/marketing/`

Cross-cutting marketing research and tool comparisons that apply across multiple businesses (not specific to any one brand).
Required tag: `marketing`

### `documents/system/`

All Molly and Clawdbot workspace files: identity, soul, memory, tools, agent configs, cron job references, second-brain sync files, and system SOPs. These are generated/maintained by Molly.
Required tag: `system`

## Frontmatter Requirements

Every document must have YAML frontmatter with at minimum:

```yaml
---
title: "Document Title"
tags: [required-folder-tag, optional-type-tag]
created: "2026-03-02T12:00:00+00:00"
updated: "2026-03-02T12:00:00+00:00"
---
```

### Tag Convention

Each file must include the **folder tag** matching its location:

| Folder                          | Required Tag         |
| ------------------------------- | -------------------- |
| `documents/dirt-roamers/`       | `dirt-roamers`       |
| `documents/anchor-staff/`       | `anchor-staff`       |
| `documents/partner-with-mateo/` | `partner-with-mateo` |
| `documents/rank-n-soar/`        | `rank-n-soar`        |
| `documents/mr-mateo-moore/`     | `mr-mateo-moore`     |
| `documents/marketing/`          | `marketing`          |
| `documents/system/`             | `system`             |
| `journals/`                     | `journal`            |
| `briefs/`                       | `brief`              |

### Type Tags (optional)

Add one or more type tags alongside the folder tag:

| Type                         | Tag              |
| ---------------------------- | ---------------- |
| Standard Operating Procedure | `sop`            |
| Research document            | `research`       |
| Guide / how-to               | `guide`          |
| Email sequence               | `email-sequence` |
| Script                       | `script`         |
| General notes                | `notes`          |
| Business plan                | `business-plan`  |
| Template                     | `template`       |
| Social media content         | `social-media`   |
| Newsletter                   | `newsletter`     |
| Ideas / brainstorm           | `ideas`          |

### YAML Safety Rules

> ⚠️ `gray-matter` uses `js-yaml` in YAML 1.1 mode, which auto-converts certain scalar values.

```yaml
# ✅ SAFE — always quote timestamps and dates
created: "2026-03-02T12:00:00+00:00"
date: "2026-03-02"

# ✅ SAFE — tags are plain lowercase hyphenated strings
tags: [dirt-roamers, research, guide]

# ❌ DANGEROUS — unquoted timestamps parse as Date objects
created: 2026-03-02T12:00:00+00:00

# ❌ DANGEROUS — numeric/date-like tags cause TypeError
tags: [notes, 2026, 2026-03-02]
```

## Adding New Documents

1. Place the file in the correct subfolder (e.g. `documents/dirt-roamers/my-doc.md`)
2. Add valid YAML frontmatter with the required folder tag
3. Commit and push → Vercel auto-deploys in ~60 seconds
4. Document appears in the UI automatically (no code changes needed)

## Adding a New Folder / Category

1. Create the subfolder under `documents/` (e.g. `documents/new-category/`)
2. Add the new folder tag to `CATEGORIES` in `src/app/page.tsx`
3. No changes to `src/lib/documents.ts` needed — it recursively scans all subdirectories
