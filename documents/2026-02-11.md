# 2026-02-11 Session Notes

## Model Provider Setup & Auth Architecture

Today we configured multiple AI model providers and established best practices for API key management.

### Providers Configured
- **Anthropic** (Claude Sonnet 4.5, Opus 4.5) - Default
- **xAI** (Grok 3, Grok 3 Mini)
- **Moonshot** (Kimi K2, Kimi K2 Turbo) - Cheapest option
- **OpenAI** (GPT-4o) - Out of budget currently

### Auth Architecture Established

**Primary auth store:** `~/.clawdbot/agents/main/agent/auth-profiles.json`
- Stores all API keys with usage tracking
- Tracks lastUsed, errorCount per provider
- Supports multi-account (e.g., `anthropic:personal`, `anthropic:work`)
- Format:
```json
{
  "version": 1,
  "profiles": {
    "provider:default": {
      "type": "api_key",
      "provider": "provider-name",
      "key": "actual-api-key"
    }
  },
  "lastGood": { "provider": "provider:default" },
  "usageStats": { "provider:default": { "lastUsed": 0, "errorCount": 0 } }
}
```

**Fallback:** `~/.clawdbot/.env`
- Standard environment variables
- Used if auth-profiles.json doesn't have the key
- Format: `PROVIDER_API_KEY=key-value`

### Key Behavior: models.json Auto-Generation

**Important:** The gateway auto-generates `~/.clawdbot/agents/main/agent/models.json` on startup!

- Source: `clawdbot.json` → `models.providers` section
- Behavior: Env var references like `${MOONSHOT_API_KEY}` get RESOLVED to actual values
- Result: models.json always contains the actual API key (not the reference)
- Cannot prevent this - it's by design
- Security: File has 600 permissions (owner only) - same as other credential files

### Cost Optimization Strategy

Mateo wants intelligent model switching to save money:
- **Simple tasks** → Kimi K2 ($0.60/$2.50 per 1M tokens)
- **Normal work** → Sonnet 4.5 ($3/$15 per 1M tokens) - DEFAULT
- **Complex reasoning** → Opus 4.5 ($15/$75 per 1M tokens)
- **Alternative/speed** → Grok 3 (~$3/$15 per 1M tokens)

### Files That Should Contain API Keys

Only these files should have actual API keys:
1. `~/.clawdbot/.env` - Fallback env vars
2. `~/.clawdbot/agents/main/agent/auth-profiles.json` - Primary auth
3. `~/.clawdbot/agents/main/agent/models.json` - Auto-generated cache (unavoidable)
4. Session logs in `~/.clawdbot/agents/main/sessions/` - Chat history

### Backups Location

Clean backups (no keys): `~/.clawdbot/defaults-original/`
- `.env.original` - Minimal .env template
- `clawdbot.json.original` - Minimal config template
