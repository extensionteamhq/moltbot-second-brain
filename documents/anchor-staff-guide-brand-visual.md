# How to Create a Brand Visual Style Guide
## A Complete Framework for Defining Your Brand's Look and Feel

### What Is a Brand Visual Style Guide?

A Brand Visual Style Guide (also called Brand Identity Guidelines or Brand Book) is a document that defines the visual elements of your brand—logo, colors, typography, imagery, and design patterns. It ensures every visual touchpoint looks cohesive and professional.

**Why It Matters:**
- Consistent visuals build recognition and trust
- Anyone (internal or external) can create on-brand materials
- Prevents "design drift" where materials slowly become inconsistent
- Protects the investment you've made in your brand identity

**Who Uses This Guide:**
- Graphic designers (internal and freelance)
- Marketing team members
- Web developers
- Print vendors
- Signage companies
- Social media managers
- External agencies and partners

---

## Section 1: Brand Overview

### What to Include

Context about the brand to help designers understand the "why" behind the visual choices.

### 1.1 Brand Story Summary

**What It Is:** A brief overview of who you are, what you do, and what you stand for.

**Deliverable Format:**
> **About [Brand Name]**
> 
> [2-3 paragraph summary including: what the company does, who it serves, the core value proposition, and the brand personality. This helps designers understand the spirit behind the visual identity.]
>
> **Our Mission:** [One sentence]
>
> **Our Vision:** [One sentence]
>
> **Our Values:** [3-5 core values]

**Why Include This:**
Visual choices should reinforce brand strategy. A designer who understands you're "approachable experts" will make different choices than one designing for "luxury exclusivity."

### 1.2 Visual Identity Principles

**What It Is:** The overarching philosophy guiding all visual decisions.

**Deliverable Format:**

> **Our visual identity is built on these principles:**
>
> 1. **Clarity over complexity** — Our designs are clean and easy to understand
> 2. **Professional yet approachable** — We're serious about results but not stuffy
> 3. **Consistent but flexible** — Core elements stay the same; application adapts to context

## Section 2: Logo

### What to Include

Everything someone needs to use your logo correctly in any situation.

### 2.1 Primary Logo

**What It Is:** The main, preferred version of your logo.

**Deliverable Format:**

- **Full-color version** (for light backgrounds)
- **High-resolution files** (vector formats: .AI, .EPS, .SVG; plus raster: .PNG at various sizes)
- **Description:** When and where to use the primary logo

**Specifications to Include:**
- File formats available
- Minimum size (smallest it can be displayed while remaining legible)
- Clear space requirements (minimum padding around the logo)

### 2.2 Logo Variations

**What It Is:** Alternate versions for different use cases.

**Deliverable Format:**

| Variation | Use Case | File Names |
|-----------|----------|------------|
| **Full color (light bg)** | Default use on white/light backgrounds | logo-full-color.svg |
| **Full color (dark bg)** | Use on dark/colored backgrounds | logo-full-color-dark.svg |
| **One-color (black)** | When color printing isn't available | logo-black.svg |
| **One-color (white)** | For use on dark backgrounds, photos | logo-white.svg |
| **Icon only** | Social media avatars, favicons, small spaces | logo-icon.svg |
| **Social media (1:1)** | Profile pictures, square format posts | logo-social-1x1.png |
| **Horizontal lockup** | Wide spaces (email headers, website nav) | logo-horizontal.svg |
| **Stacked/vertical lockup** | Tall spaces (signage, narrow columns) | logo-stacked.svg |

**For Each Variation, Provide:**
- Visual example
- When to use it
- File location/name

### 2.3 Clear Space & Minimum Size

**What It Is:** Rules for how much breathing room the logo needs and how small it can go.

**Deliverable Format:**

**Clear Space:**
> The logo must have a minimum clear space equal to [X] on all sides, where [X] is defined as [measurement—often the height of a letter or specific pixel/inch amount].
>
> [Include diagram showing clear space around logo]

**Minimum Size:**
> - **Print:** No smaller than [X inches/mm] wide
> - **Digital:** No smaller than [X pixels] wide
>
> Below these sizes, use the icon-only version.

### 2.4 Logo Misuse

**What It Is:** Examples of what NOT to do with the logo.

**Deliverable Format:**

Show visual examples of incorrect usage with ✗ marks:

- ✗ Do not stretch or distort the logo
- ✗ Do not rotate the logo
- ✗ Do not change the logo colors
- ✗ Do not add effects (shadows, outlines, gradients)
- ✗ Do not place on busy backgrounds without sufficient contrast
- ✗ Do not rearrange logo elements
- ✗ Do not use outdated versions
- ✗ Do not recreate or redraw the logo

## Section 3: Color Palette

### What to Include

All brand colors with exact specifications for any medium.

### 3.1 Primary Colors

**What It Is:** The main colors that define your brand (typically 2-3).

**Deliverable Format:**

| Color Name | Use Case | HEX | RGB | CMYK | Pantone |
|------------|----------|-----|-----|------|---------|
| **[Name] Blue** | Primary brand color, CTAs, headers | #1A56DB | 26, 86, 219 | 88, 61, 0, 14 | PMS 2728 C |
| **[Name] Dark** | Text, backgrounds, accents | #1E1E1E | 30, 30, 30 | 0, 0, 0, 88 | Black 7 C |
| **White** | Backgrounds, reversed text | #FFFFFF | 255, 255, 255 | 0, 0, 0, 0 | — |

**Include For Each Color:**
- **HEX:** For web/digital (#1A56DB)
- **RGB:** For digital/screen (26, 86, 219)
- **CMYK:** For print (88, 61, 0, 14)
- **Pantone:** For premium print/merchandise (PMS 2728 C)

### 3.2 Secondary Colors

**What It Is:** Supporting colors for variety and hierarchy (typically 3-5).

**Deliverable Format:**

Same format as primary colors, plus:
- When to use each secondary color
- Proportion guidelines (e.g., "Secondary colors should not exceed 20% of any design")

### 3.3 Accent/Utility Colors

**What It Is:** Functional colors for UI elements, alerts, and specific purposes.

**Deliverable Format:**

| Color | Purpose | HEX |
|-------|---------|-----|
| Success Green | Positive actions, confirmations | #22C55E |
| Warning Yellow | Caution, attention needed | #EAB308 |
| Error Red | Errors, destructive actions | #EF4444 |
| Info Blue | Informational messages | #3B82F6 |
| Muted Gray | Disabled states, secondary text | #9CA3AF |

### 3.4 Color Usage Guidelines

**What It Is:** Rules for how colors work together.

**Deliverable Format:**

**Color Hierarchy:**
> - Primary Blue: Main CTAs, key headlines, brand moments
> - Dark: Body text, navigation, secondary elements
> - White: Backgrounds, cards, breathing room
> - Secondary colors: Accents, icons, supporting graphics

**Accessibility:**
> All text must meet WCAG 2.1 AA contrast requirements:
> - Normal text: 4.5:1 contrast ratio minimum
> - Large text (18pt+): 3:1 contrast ratio minimum
>
> [Include approved color combinations for text/background]

**Color Don'ts:**
- ✗ Do not use colors not in this palette
- ✗ Do not use primary blue for large background areas
- ✗ Do not use low-contrast color combinations for text

## Section 4: Typography

### What to Include

All fonts, weights, and how to use them.

### 4.1 Font Families

**What It Is:** The typefaces that define your brand.

**Deliverable Format:**

**Primary Font: [Font Name]**
> Used for: Headlines, titles, key statements
> 
> Weights available: Light, Regular, Medium, Bold, Black
>
> [Visual example showing the font at various weights]
>
> **Source:** [Google Fonts / Adobe Fonts / Purchase link]
> **License:** [Open source / Commercial license required]

**Secondary Font: [Font Name]**
> Used for: Body text, paragraphs, general reading
>
> Weights available: Regular, Medium, SemiBold
>
> [Visual example]
>
> **Source:** [Link]

**Fallback Fonts:**
> If brand fonts are unavailable, use these system fonts:
> - Headlines: Arial Black, Helvetica Bold
> - Body: Arial, Helvetica, sans-serif

### 4.2 Type Scale & Hierarchy

**What It Is:** Specific sizes and styles for different text levels.

**Deliverable Format:**

**Web/Digital:**

| Level | Font | Weight | Size | Line Height | Use Case |
|-------|------|--------|------|-------------|----------|
| H1 | Primary | Bold | 48px | 1.2 | Page titles |
| H2 | Primary | Bold | 36px | 1.25 | Section headers |
| H3 | Primary | SemiBold | 24px | 1.3 | Subsection headers |
| H4 | Primary | Medium | 20px | 1.4 | Card titles |
| Body | Secondary | Regular | 16px | 1.6 | Paragraph text |
| Small | Secondary | Regular | 14px | 1.5 | Captions, meta |
| Button | Primary | SemiBold | 16px | 1 | CTAs |

**Print:**
[Similar table with print-appropriate sizes in points]

### 4.3 Typography Rules

**What It Is:** Guidelines for text formatting and style.

**Deliverable Format:**

**General Rules:**
- Maximum line length: 65-75 characters for readability
- Paragraph spacing: 1.5x the line height
- Headlines: Sentence case ("How to build your brand")
- Never use all caps for more than 3 words
- Avoid justified text; use left-aligned

**Emphasis:**
- **Bold** for strong emphasis
- *Italic* for titles, quotes, or light emphasis
- Avoid underlines (except for links)
- Avoid using color alone for emphasis

**Don'ts:**
- ✗ Don't use fonts not in this guide
- ✗ Don't stretch or compress type
- ✗ Don't use decorative fonts
- ✗ Don't mix more than 2 font families in one design

**Deliverable Format:**

**Web:**

> Margins: 16px (mobile), 24px (tablet), 48px (desktop)

> [Columns, margins, and bleeds for common print formats]

## Section 9: Implementation

### 9.1 How to Use This Guide

**For Designers:**
- Download all assets before starting any project
- Reference this guide for every design decision
- When in doubt, prioritize clarity and consistency

**For Vendors:**
- Share relevant sections (not the entire guide if unnecessary)
- Provide specific assets for the job
- Request proofs for review before final production

**For Quality Control:**
- Create a checklist from this guide
- Review all materials before publishing/printing
- Update the guide as the brand evolves

### 9.2 Brand Compliance Checklist

- [ ] Correct logo version for the application
- [ ] Logo has proper clear space
- [ ] Colors match exact specifications
- [ ] Fonts are correct (not substituted)
- [ ] Typography follows hierarchy rules
- [ ] Images match photography style guidelines
- [ ] Layout follows grid and spacing system
- [ ] All text is legible and accessible
